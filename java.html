Java Example

A complete, end-to-end walkthrough of supervising a long-running Java job with AutoCheck — config, run, resume, and what ends up on disk.

The config

Save this as config.yaml next to your compiled Java program. execution_mode: hpc and language: java tell AutoCheck to track the variables you list under hpc.tracked_states, using each variable's name exactly as it appears in your source.

yaml
system:
  execution_mode: hpc
  fram_schd: general
  run_id: run1
  language: java

checkpoint:
  method: time
  interval: 300             # seconds between checkpoints
  max_session_time: 900     # 15 minutes total
  safety_buffer_seconds: 3
  keep_last: 3
  save_dir: checkpoints
  program_path: Testing_phase/HPCJob.java

hpc:
  tracked_states:
    - name: progress_percent
      type: float
      source: jdwp
    - name: iteration_count
      type: int
      source: jdwp

Before you run it

Make sure your JDK supports remote debugging — standard on any modern JDK install, no extra setup needed on your end. Your progress_percent and iteration_count variables just need to exist in your program; you don't add any logging, annotations, or AutoCheck imports to HPCJob.java.

Running it

bash
autocheck run --config config.yaml

To check the config for errors first, without starting the job:

bash
autocheck run --config config.yaml --validate-only

What you'll see in save_dir

HPC jobs don't have an epoch or step counter, so checkpoints are named by timestamp instead:

text
checkpoints/
├── checkpoint_20260629-141200/
├── checkpoint_20260629-141700/
├── checkpoint_20260629-142200/
├── session_info.json
└── .autocheck.pid

Each checkpoint holds the values of progress_percent and iteration_count captured at that moment. Only the 3 most recent checkpoints are kept, per keep_last: 3.

Stopping and resuming

Stop the job gracefully — a final checkpoint is saved first:

bash
autocheck stop --config config.yaml

Resume from the latest checkpoint:

bash
autocheck resume --config config.yaml

progress_percent and iteration_count are restored to their saved values and your program continues from there. See Language Support — Java for requirements, and the schema reference for every field under hpc.tracked_states.
