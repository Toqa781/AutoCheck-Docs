TraceLayer: Non-Invasive Code Inspection

Uses sys.settrace to inspect user locals without code changes — the mechanism behind AutoCheck's "zero modification" promise.

Relevant source files tracing_how.zip

How it works

Python's sys.settrace lets a process register a callback that fires on every line execution, function call, and return within traced frames. TraceLayer installs such a callback scoped to the user's training function, then reads local variables, model, optimizer, loss, epoch counters, directly out of the frame object as it executes.

Read-only observation

TraceLayer never injects code or mutates the user's local variables — it only reads them to feed StateTracker.

Tradeoffs

Trace-based inspection carries a small, measurable overhead per traced line compared to fully manual instrumentation. AutoCheck accepts this cost in exchange for requiring no changes to the user's training code, which is the project's core design constraint.
