Zero code modification

No imports inside your program, no manual save() calls. AutoCheck inspects your script's running state from the outside.

Sidecar execution model

The Autonomous Controller supervises from the main thread while your script runs untouched in a dedicated daemon thread.

Framework-and-Language agnostic

First-class support for Python, Java, C, and C++ programs. And deeper framework integration currenlty covers PyTorch, Keras/TensorFlow, and Scikit-Learn — with a core engine that stays decoupled from any one of them.

Core design philosophy

AutoCheck is designed around three core principles. First, it minimizes changes to user applications by requiring only minimal integration. Second, it operates independently from the application workflow, allowing checkpointing and monitoring to occur without disrupting execution. Finally, its modular architecture enables support for multiple execution environments and programming languages while providing a consistent user experience.

What "sidecar" means here

The AutonomousController runs in the main thread and owns the checkpoint lifecycle. Your training script runs in a dedicated daemon thread, fully isolated from the save/load machinery happening around it.

Mapping natural language to code entities

To bridge high-level concepts and implementation, here's how the system's conceptual roles map onto actual classes.

The Brain

The Bridge
