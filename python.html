Python Example

A complete, end-to-end walkthrough of supervising a PyTorch training script with AutoCheck — config, run, resume, and what ends up on disk.

The config

Save this as config.yaml next to your training script. execution_mode: ml and language: python tell AutoCheck this is a Python training job — everything else AutoCheck discovers from your script automatically.

yaml
system:
  execution_mode: ml
  fram_schd: pytorch       # or tensorflow
  run_id: run1
  language: python

ml_model:
  name: deepNN

checkpoint:
  method: time
  interval: 180             # seconds between checkpoints
  max_session_time: 7200    # 2 hours total
  safety_buffer_seconds: 60
  keep_last: 3
  save_dir: checkpoints/DeepNN
  program_path: Testing_phase/deepNN.py

notify:
  email: you@example.com
  on_checkpoint: true
  on_failure: true

Running it

No changes to deepNN.py are needed — write your training loop exactly as you normally would. Then start the job:

bash
autocheck run --config config.yaml

AutoCheck launches deepNN.py under supervision and takes a snapshot every 180 seconds. If you'd like to confirm your config is valid before actually launching training, use:

bash
autocheck run --config config.yaml --validate-only

What you'll see in save_dir

After a few checkpoints, checkpoints/DeepNN/ contains:

text
checkpoints/DeepNN/
├── checkpoint_00000180.pt
├── checkpoint_00000360.pt
├── checkpoint_00000540.pt
├── session_info.json
└── .autocheck.pid

Only the 3 most recent .pt files are kept, per keep_last: 3 — older ones are deleted automatically as new ones are saved. Each file is a serialized state dict containing model weights, optimizer state, and training progress.

Stopping and resuming

To stop the job gracefully — saving a final checkpoint first:

bash
autocheck stop --config config.yaml

And to pick training back up from the latest checkpoint:

bash
autocheck resume --config config.yaml

Model weights, optimizer state, and epoch/batch counters are restored automatically, and training continues as if it had never stopped. See Resume Jobs for resuming from a specific checkpoint instead of the latest one, and Snapshots for exactly what gets captured at each save.
