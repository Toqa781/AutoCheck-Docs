Glossary

Definitions for the core terms used throughout this documentation.

Terms

TermDefinition
CheckpointA saved snapshot of a job's complete execution state — model weights, optimizer state, and metadata — written to a structured directory.
Sidecar executionThe pattern where a supervising process (the controller) runs alongside the user's script in a separate thread, without modifying it.
State trackerThe shared object holding the current ground truth of a running job, read by the controller to evaluate trigger conditions.
TriggerA condition, time interval, session limit, or OS signal, that causes the controller to initiate a checkpoint save.
Interval triggerA trigger that fires on a fixed wall-clock cadence, configured via interval_minutes.
Session time limitThe configured maximum runtime for a single session, after which a final checkpoint is saved and the job stops.
ResumeRestarting a job from its most recently saved checkpoint rather than from the beginning.
TraceLayerThe component that uses Python's sys.settrace to read training-loop variables without requiring code changes.
SignalLayerThe component that intercepts OS signals such as SIGTERM and SIGUSR1 to trigger a final checkpoint before a process is killed.
Framework agnosticDesigned so the core engine doesn't depend on any one ML framework; PyTorch, Keras, and Scikit-Learn are each supported through dedicated checkpoint managers.
