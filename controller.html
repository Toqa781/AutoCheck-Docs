Controller and Execution Engine

The AutonomousController manages the checkpoint loop, evaluates trigger conditions, and handles graceful shutdowns — the system's decision-making core.

Relevant source files main.py Description.md

The brain of the system

AutonomousController is the component every other layer ultimately answers to. It owns the main thread, polls the shared StateTracker on a fixed cadence, and decides — based on configured policy — when a save is due, when a load should happen, and when the entire job should stop.

Decides
when to save

Evaluated against interval_minutes on every controller tick.

Decides
when to load

On resume, delegates to CheckpointManager.load() before training restarts.

Decides
when to stop

Triggered by session_time_limit or an intercepted SIGTERM.

Coordinates
graceful shutdown

Ensures a final checkpoint is written before the daemon thread is torn down.

The control loop, conceptually

On each tick, the controller reads the current state, checks it against active trigger policies, and acts if any policy condition is met. Full trigger semantics are documented in Checkpoint Loop and Trigger Logic.

Polling, not interrupting your code

The controller never pauses your training loop to ask permission. It reads whatever StateTracker currently holds and acts independently, pausing the daemon thread only for the duration of an actual checkpoint write.
