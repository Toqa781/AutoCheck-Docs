SignalLayer: HPC Scheduler Signal Handling

Intercepts OS signals such as SIGUSR1 and SIGTERM for HPC safety, converting scheduler kill warnings into final-checkpoint triggers.

Relevant source files layers/SignalLayer.py

Signal registration

SignalLayer installs handlers for SIGTERM and SIGUSR1 at process start, before the user's training script begins executing. Both handlers route into the same final-checkpoint code path used by an interval or session-limit trigger.

text
scheduler sends SIGUSR1 (warning, ~5 min before kill)
  → SignalLayer intercepts
  → AutonomousController triggers checkpoint save
  → training continues if time remains

scheduler sends SIGTERM (final kill signal)
  → SignalLayer intercepts
  → AutonomousController triggers final checkpoint save
  → process exits cleanly
Not every scheduler sends SIGUSR1

Whether a warning signal is available before SIGTERM depends on the scheduler's configuration. Where it isn't, session_time_limit in config.yaml is the safety net.
