Configuration Validation

AutoCheck reads and validates your config.yaml before launching the job. Problems are caught immediately at startup not hours into a run.

What gets checked

CheckApplies toWhat happens on failure
system.language is present and validAll jobsAutoCheck stops immediately there is no default.
system.execution_mode is ml or hpcAll jobsAutoCheck stops and reports the unrecognised value.
checkpoint.save_dir is writableAll jobsAutoCheck stops an unwritable path discovered mid-run would silently lose your checkpoints.
checkpoint.program_path existsAll jobsAutoCheck stops and reports the missing file path.
system.fram_schd is pytorch or tensorflowML jobsAutoCheck stops and reports the invalid value.
hpc.tracked_states is present and non-emptyHPC jobsAutoCheck stops — without tracked states there is nothing to checkpoint.
Each tracked state has name, type, and sourceHPC jobsAutoCheck stops and identifies the incomplete entry.
notify.email is a valid address (if notify is present)Jobs with notificationsAutoCheck stops rather than running silently with broken notifications.
Defaults are applied silently

Optional keys you leave out like safety_buffer_seconds or keep_last — are filled in automatically. You only see an error for keys that are truly required and missing.

Common errors and fixes

Error messageLikely causeFix
system.language is requiredlanguage key missing from the system section.Add language: python (or java, c, cpp) under system:.
Unknown execution_modeexecution_mode is misspelled or set to an unsupported value.Use ml or hpc exactly.
save_dir is not writableThe checkpoint folder doesn't exist or AutoCheck doesn't have write permission.Create the folder first, or point save_dir to a location you own.
program_path not foundThe path to your program is wrong or the file doesn't exist yet.Check the path is correct relative to where you run AutoCheck, or use an absolute path.
tracked_states is emptyThe hpc section exists but has no entries under tracked_states.Add at least one variable entry with name, type, and source.
Invalid email addressThe notify.email value is malformed.Double-check the email address format in your config.
