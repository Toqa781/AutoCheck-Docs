ConfigManager Internals

Loads and parses the YAML configuration file and exposes its values to the rest of the system through a typed accessor.

Relevant source files test_config_manager.py config.yaml error.py

Responsibilities

  • Read the YAML file at the path given to RunnerScript
  • Validate that required keys are present and of the expected type
  • Apply defaults for optional keys that were omitted
  • Expose values to AutonomousController, StateTracker, and CheckpointManager implementations

Validation failures

Malformed or incomplete configuration raises a custom exception from error.py rather than failing silently or falling back to an unexpected default for anything safety-relevant, such as storage_path.

Fail fast on config errors

Catching a misconfigured storage_path before training starts is far cheaper than discovering it hours into a run when the first checkpoint write fails.
