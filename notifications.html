Notification Service

Sends events when checkpoints are saved, jobs fail, or jobs complete, so a long-running job doesn't require constant manual monitoring.

Relevant source files logger.py

Why notifications

HPC and long-running ML jobs are, by definition, unattended for long stretches. The notification service surfaces the events that matter, a checkpoint was saved, a job resumed, a job failed, without requiring someone to tail logs in real time.

Notification configuration and events

EventFires when
on_saveA checkpoint write completes successfully.
on_resumeA job loads a checkpoint and resumes training.
on_failureA checkpoint write or load operation raises an exception.
Built on the same logging infrastructure

Notification events are emitted through the structured logger described in Logging Infrastructure, so they can be routed wherever your existing log pipeline already sends output.
