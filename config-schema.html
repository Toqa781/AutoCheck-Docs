Configuration Schema Reference

Complete reference for every key recognized under checkpointing and training in config.yaml.

Relevant source files config.yaml

checkpointing

KeyTypeDefaultDescription
enabledbooltrueMaster switch. When false, the controller still runs but never writes a checkpoint.
interval_minutesint30Minutes between automatic interval-triggered checkpoints.
session_time_limitint120Minutes after which the job stops and writes a final checkpoint, matching an HPC scheduler's allocation.
storage_pathpath./checkpointsRoot directory for the structured checkpoint layout (stage/epoch-batch/).
frameworkenum—Either pytorch or tensorflow. Selects which CheckpointManager implementation is instantiated.

training

KeyTypeDefaultDescription
epochsint—Total planned epochs, used to validate resume state and for progress display.
batch_sizeint—Batch size used by the dataloader. Recorded in checkpoint metadata for sanity checks on resume.

Full example

yaml
checkpointing:
  enabled: true
  interval_minutes: 30
  session_time_limit: 120
  storage_path: ./checkpoints
  framework: pytorch

training:
  epochs: 100
  batch_size: 64
