Configuration Schema Reference

Every key recognized in config.yaml, with its type, default, and a plain-English description of what it controls.

system

Top-level identity and mode settings. All keys in this section are required.

KeyTypeValuesDescription
execution_modeenumml · hpcWhether AutoCheck is supervising an ML training job or an HPC compute job. Controls which sections are read.
fram_schdstringpytorch · tensorflow · generalThe framework or scheduler in use. Set to pytorch or tensorflow for ML jobs; general for HPC jobs.
run_idstringany stringA label for this run, used to name checkpoint folders and log entries. Choose something meaningful, e.g. experiment-3.
languageenumpython · c · cpp · javaThe language of your target program. AutoCheck uses this to select the correct attachment and state-capture mechanism. Required — no default.

checkpoint

Controls when, where, and how many checkpoints are saved. Applies to both ML and HPC jobs.

KeyTypeDefaultDescription
methodenumtimeHow checkpointing is triggered. Currently time — saves on a fixed time interval.
intervalint—Seconds between automatic checkpoints. For example, 300 saves every 5 minutes.
max_session_timeint—Total seconds AutoCheck will run before stopping and saving a final checkpoint. Set this to match your HPC scheduler's wall-clock limit, minus some margin.
safety_buffer_secondsint60How many seconds before max_session_time the final checkpoint is triggered. Gives the save operation time to finish before the scheduler cuts the job.
keep_lastint3Number of most-recent checkpoints to keep on disk. Older ones are deleted automatically.
save_dirpath—Folder where checkpoint files are saved. Must be writable. AutoCheck checks this at startup.
program_pathpath—Path to your program's entry point — the .py, .java, or compiled binary that AutoCheck will launch and supervise.

ml_model

ML jobs only (execution_mode: ml). Metadata about the model being trained.

KeyTypeDefaultDescription
namestring—A name for your model, e.g. deepNN. Used in log output and checkpoint folder naming.

hpc

HPC jobs only (execution_mode: hpc). Defines the program variables AutoCheck will capture at each checkpoint.

KeyTypeDescription
tracked_stateslistA list of program variables to capture. Each entry has three fields: name (the variable name in your program), type (e.g. float, int), and source (how AutoCheck reads it — currently jdwp for Java).

Example with multiple tracked variables:

yaml
hpc:
  tracked_states:
    - name: progress_percent
      type: float
      source: jdwp
    - name: iteration_count
      type: int
      source: jdwp

notify

Optional for both job types. Sends email alerts when checkpoints are saved or when the job fails.

KeyTypeDefaultDescription
emailstring—Email address to send notifications to.
on_checkpointboolfalseSend an email every time a checkpoint is successfully saved.
on_failureboolfalseSend an email if the job crashes or a checkpoint write fails.
