Configuration Schema Reference

Complete reference for every key recognized in config.yaml, across all five top-level sections.

language

Selects which provider layer AutoCheck instantiates. This key is always required.

KeyTypeDefaultDescription
languageenum—One of python, c, cpp, or java. Determines the layer used to attach to, trace, and restore the target process. Required — no default.

checkpointing

KeyTypeDefaultDescription
enabledbooltrueMaster switch. When false, the controller still runs but never writes a checkpoint.
interval_minutesint30Minutes between automatic interval-triggered checkpoints.
session_time_limitint120Minutes after which the job stops and writes a final checkpoint, matching an HPC scheduler's wall-clock allocation.
storage_pathpath./checkpointsRoot directory for the structured checkpoint layout (stage/epoch-batch/). Must be writable. Validated at startup — a bad path causes an immediate error rather than a silent failure hours into a run.
frameworkenum—pytorch or tensorflow. Selects the CheckpointManager implementation. Only applies when language: python. Omit for java, c, or cpp.

training

Python ML jobs only. Used for progress display and resume sanity checks.

KeyTypeDefaultDescription
epochsint—Total planned epochs, used to validate resume state and for progress display.
batch_sizeint—Batch size used by the dataloader. Recorded in checkpoint metadata for sanity checks on resume.

java

Required when language: java. Configures how AutoCheck connects to the JVM.

AutoCheck reads state over JDWP (port 5005 by default) and writes restored values via a control socket (port 5006 by default), because JDWP returns WRITE_PROTECTED for field writes. Both ports must be free before the run starts.

KeyTypeDefaultDescription
main_classstring—Fully-qualified Java class containing main() — for example HPCJob or com.example.Simulation. Required.
classpathpath.Classpath passed to java -cp. Use : (Linux/macOS) or ; (Windows) to separate multiple entries.
jdwp_portint5005Port for JDWP debug attachment (state reading). Must be in the range 1024–65535 and must not equal control_port.
control_portint5006Port for the write-agent control socket (state injection). Must be in the range 1024–65535 and must not equal jdwp_port.
Port conflicts

If jdwp_port (default 5005) is held by an orphaned JVM process from a previous session, AutoCheck will attempt to kill that process automatically at startup. If the port is still unavailable afterward, the run fails immediately with a clear error rather than hanging. Use jdwp_port and control_port to remap to free ports if the defaults are already in use by another application.

hpc

Required for C, C++, and Java HPC jobs. Activates session-limited execution and pre-emptive checkpointing to match HPC scheduler wall-clock limits.

KeyTypeDefaultDescription
enabledboolfalseActivates HPC mode. When true, AutoCheck respects checkpointing.session_time_limit and performs a final pre-emptive checkpoint before the allocation expires.
schedulerenum—Set to jdwp for Java jobs. Controls which mechanism is used to capture state on pre-emption. Omit for C and C++ (ptrace is used automatically).

Full example — Python ML

yaml
language: python

checkpointing:
  enabled: true
  interval_minutes: 30
  session_time_limit: 120
  storage_path: ./checkpoints
  framework: pytorch

training:
  epochs: 100
  batch_size: 64

Full example — Java HPC

yaml
language: java

java:
  main_class: HPCJob
  classpath: ./build
  jdwp_port: 5005
  control_port: 5006

hpc:
  enabled: true
  scheduler: jdwp

checkpointing:
  enabled: true
  interval_minutes: 30
  session_time_limit: 120
  storage_path: ./checkpoints
