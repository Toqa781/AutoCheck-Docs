Getting Started

Install AutoCheck, write a minimal config file, and run your first supervised job — without touching a single line of your training loop.

Relevant source files README.md requirements.txt config.yaml main.py

Requirements

AutoCheck targets standard Python ML environments. There's no custom runtime, no daemon to install system-wide — it runs as a regular Python process alongside your script.

RequirementVersionNotes
Python3.8+Required for the typing and threading features used by the controller.
PyTorchany recentRequired only if using PyTorchCheckpointManager.
TensorFlowany recentRequired only if using KerasCheckpointManager.
PyYAMLany recentUsed by ConfigManager to parse config.yaml.

Installation

Install the framework-specific dependency you need alongside PyYAML, then clone the repository.

bash 
# for PyTorch projects
pip install torch pyyaml

# for TensorFlow / Keras projects
pip install tensorflow pyyaml
bash 
git clone https://github.com/habibaaymannn/AutoCheck.git
cd AutoCheck
pip install -r requirements.txt

Write a minimal config

Every run is controlled by a single YAML file. At minimum, point it at a framework and a storage path — everything else has a sane default.

yaml 
checkpointing:
  enabled: true
  interval_minutes: 30        # save a checkpoint every 30 minutes
  session_time_limit: 120    # stop and save after 120 minutes
  storage_path: ./checkpoints
  framework: pytorch      # pytorch or tensorflow

training:
  epochs: 100
  batch_size: 64

See the full Configuration Schema Reference for every available key.

Run your first job

Wrap your existing training call with RunnerScript — no changes inside the loop itself.

python 
from autocheck import RunnerScript

# Point to your model, dataloader, and config — no changes to your training loop
runner = RunnerScript(config_path="config.yaml")
runner.train_with_checkpoints(model, dataloader, optimizer)

If the job is interrupted, by a time limit, a signal, or a crash, resume from the command line:

bash 
python run.py --resume --config config.yaml
Resume is automatic

AutoCheck detects the most recent checkpoint under storage_path and resumes training from the last saved epoch and batch — no manual bookkeeping required.
