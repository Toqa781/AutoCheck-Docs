Getting Started

Install AutoCheck, write a minimal config file, and run your first supervised job — without touching a single line of your training loop.

Requirements

AutoCheck runs as a regular Python process alongside your script — no custom runtime, no daemon to install system-wide.

RequirementVersionNotes
Python3.8+Required for the typing and threading features used by the controller.
PyTorchany recentRequired only for ML jobs using fram_schd: pytorch.
TensorFlowany recentRequired only for ML jobs using fram_schd: tensorflow.
PyYAMLany recentUsed by ConfigManager to parse config.yaml.
GDBany recentRequired only for C and C++ HPC jobs.
JDK with JDWPany modern JDKRequired only for Java HPC jobs.

Add AutoCheck to your PATH

After extracting the zip, add the folder containing autocheck.exe to your system's environment variables so you can run autocheck from any directory.

  1. Open Start → Search → "Edit the system environment variables"
  2. Click Environment Variables…
  3. Under System variables, select Path and click Edit
  4. Click New and paste the full path to the folder containing autocheck.exe — e.g. C:\Users\you\AutoCheck
  5. Click OK on all dialogs, then open a new terminal to apply the change

Verify the setup by running:

bash
autocheck help

You should see the full command reference:

text
AutoCheck - Available Commands
===============================
  run     Start a fresh run with checkpointing.
            --config PATH          Path to the YAML config file (required)
            --mode  ml|hpc         Override execution_mode from config
            --save-dir DIR         Override checkpoint.save_dir from config
            --validate-only        Validate config and exit without launching
            --force-fresh          Start fresh even if checkpoints exist
  stop    Gracefully stop a running job and save a checkpoint.
            --config PATH          Path to the YAML config file (required)
            --mode  ml|hpc         Override execution_mode from config
            --save-dir DIR         Override checkpoint.save_dir from config
  resume  Resume from the latest (or a specific) checkpoint.
            --config PATH          Path to the YAML config file (required)
            --mode  ml|hpc         Override execution_mode from config
            --save-dir DIR         Override checkpoint.save_dir from config
            --checkpoint PATH      Resume from a specific checkpoint file
  help    Show this help message.
 
Examples:
  autocheck run    --config config.yaml
  autocheck stop   --config config.yaml
  autocheck resume --config config.yaml
  autocheck run    --config config.yaml --mode ml --save-dir ./ckpts
  autocheck run    --config config.yaml --validate-only
  autocheck resume --config config.yaml --checkpoint ./ckpts/ckpt_42.pt

Write a minimal config

Every run is controlled by a single config.yaml file. Place it in your project's working directory — AutoCheck reads it automatically, or you can pass an explicit path with --config.

yaml
system:
  execution_mode: ml
  fram_schd: pytorch       # or tensorflow
  run_id: run1
  language: python

ml_model:
  name: my_model

checkpoint:
  method: time
  interval: 1800            # seconds — save every 30 minutes
  max_session_time: 7200    # stop and save after 2 hours
  safety_buffer_seconds: 60
  keep_last: 3
  save_dir: checkpoints
  program_path: train.py

See the Configuration System for ready-to-copy full examples per language, and the Configuration Schema Reference for every available key and its default.

Run your first job

No changes to train.py are needed — write your training loop exactly as you normally would. Then start the job:

bash
autocheck run --config config.yaml

AutoCheck launches train.py under supervision and saves a checkpoint automatically every 30 minutes. If the job is interrupted — by a time limit, a scheduler signal, or a crash — resume from the latest checkpoint with:

bash
autocheck resume --config config.yaml
Resume is automatic

AutoCheck finds the most recent valid checkpoint in save_dir and restores model weights, optimizer state, and training progress automatically — no manual bookkeeping required.

What's next

Once your first job runs end to end, see Language Support if you're running a Java, C, or C++ job, the Architecture Overview for the full threading model, or Resume Jobs for resuming from a specific checkpoint rather than the latest one.
