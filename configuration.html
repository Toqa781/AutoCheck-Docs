Configuration System

All checkpointing behavior is controlled through a single YAML file — no environment variables, no scattered flags.

Relevant source files config.yaml test_config_manager.py

Single source of truth

Every parameter that affects how, when, and where AutoCheck saves state lives in one config.yaml file. This keeps a training run reproducible: the config file alongside the checkpoint directory is enough to understand exactly how that run was supervised.

yaml
checkpointing:
  enabled: true
  interval_minutes: 30
  session_time_limit: 120
  storage_path: ./checkpoints
  framework: pytorch

training:
  epochs: 100
  batch_size: 64

Top-level sections

SectionPurpose
checkpointingEverything related to save cadence, storage location, and target framework.
trainingBasic job metadata such as epoch and batch-size targets, used for progress display and resume validation.
